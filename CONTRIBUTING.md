# Contributing to ThreadMind

First off, thank you for considering contributing to ThreadMind! It's people like you that make ThreadMind such a great tool.

## How Can I Contribute?

### Reporting Bugs
- Use a clear and descriptive title for the issue.
- Describe the exact steps which reproduce the problem in as many details as possible.
- Explain which behavior you expected to see instead and why.
- **Include screenshots and logs** where possible.

### Suggesting Enhancements
- If you're suggesting an enhancement, use the Feature Request template.
- Explain why this enhancement would be useful to most ThreadMind users.
- Provide a step-by-step description of the suggested enhancement.

### Pull Requests
- Fill out the [pull request template](.github/PULL_REQUEST_TEMPLATE.md).
- Follow the existing code style and formatting.
- Ensure that your changes do not break existing functionality.
- Document any new features or changes to existing ones.

## Style Guide
- Use TypeScript and follow the project's `tsconfig.json`.
- Write clear, commented, and modular code.
- Ensure all new features have corresponding documentation in the `docs/` folder.

## Community
Join the discussion on [GitHub Discussions](discussions.md)!
